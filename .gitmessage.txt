# ==========================================
# 📝 Commit Convention (Team Standard)
# ==========================================
# [형식]
# <이모지> <타입>: <간단한 제목> (영문/한글 모두 가능)
#
# [제목 규칙]
# - 50자 이내로 간결하게
# - 첫 글자는 소문자, 마침표 금지
# - 명령형 어조 사용 (ex: "add", "update", "fix")
#
# [본문 규칙]
# - 선택 사항 (있으면 72자 이내 줄바꿈)
# - 무엇을, 왜, 어떻게 변경했는지 설명
# - 관련된 이슈나 참고사항 기록
#
# [푸터 규칙]
# - 이슈 트래킹: Closes #123, Resolves JIRA-456
# - BREAKING CHANGE: 호환성 깨지는 변경 사항 설명
#
# ==========================================
# Commit Type Guide
#
# ✨ feat:     새로운 기능 추가
# 🐛 fix:      버그 수정
# ♻️ refactor: 코드 리팩터링 (기능 변경 없음)
# 📝 docs:     문서 수정 (ex: README, API 명세)
# 💡 chore:    기타 변경 (오타, 주석, 설정 등)
# 🚚 build:    빌드, 배포 관련 파일 수정
# ✅ test:     테스트 코드 추가/수정
# 🚑 hotfix:   긴급 수정 (프로덕션 문제)
#
# ==========================================
# 예시
#
# ✨ feat: 사용자 로그인 기능 추가
#
# - JWT 기반 로그인 API 구현
# - 유효성 검증 로직 보강
#
# Closes #21
#
# 🐛 fix: 회원가입 시 이메일 중복 오류 해결
#
# - DB unique 제약 조건 누락 수정
# - Validation 모듈에 이메일 정규식 추가.
#
# Resolves JIRA-102
# ==========================================
